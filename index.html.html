DocFlow Agent is a production-grade AI documentation automation system I designed and built to demonstrate how Skills, MCP Servers, Plugins, and Agents work together in a real technical writing workflow. This is the actual system I use to manage documentation at Salesforce scale — adapted and documented here as a portfolio project.

The Four Layers — How They Work Together

Each layer plays a distinct role. Together they form a complete, autonomous documentation pipeline.

Layer 1

Skills

Reusable Claude instruction playbooks. Written once, triggered by name. The output is always consistent, always on-brand, always following our standards. I built four Skills for this project.

Layer 2

MCP Servers

Live connections between Claude and my real tools — GitHub, Slack, Google Drive, and the local filesystem. No more copy-pasting. Claude reads and writes directly.

Layer 3

Plugins

Specialized capabilities added to Claude — web search to look up current API versions, and code execution to verify that examples actually work before including them in documentation.

Layer 4

Agents

Autonomous orchestrators. I give an agent a goal — it decides which Skills and MCP connections to use at each step and executes the full workflow without me guiding it manually.

Skills — Four Reusable Instruction Playbooks

Each Skill file contains the complete instruction set, example input, and example output. These files encode years of editorial judgment into reusable AI workflows.

Skill What It Does Time Saved
release-note-generator Converts raw GUS engineering stories into customer-facing release notes — with correct classification, customer perspective, and action steps 45 min → 10 min
api-endpoint-documenter Generates complete API reference pages from engineering specs — parameter tables, tested curl examples, response examples, and error codes 2 hrs → 20 min
sme-notes-processor Transforms raw SME call notes into four outputs: documentation outline, follow-up questions, terminology list, and risk flags 60 min → 5 min
doc-consistency-auditor Audits batches of articles against a style guide across five dimensions — produces a prioritized fix list: Critical, High, Medium, Low 2 weeks → 2 days

MCP Servers — Live Connections to Real Tools

MCP (Model Context Protocol) Server connections give Claude direct read and write access to my tools in real time. No copy-pasting. Claude works with the actual files, the actual repository, the actual Slack channel.

MCP Server Connected Tool What Claude Can Do
GitHub MCP GitHub repository Read files, commit changes, open pull requests, read issues
Filesystem MCP Local docs folder Read and write local Markdown files directly
Slack MCP Slack workspace Post review notifications to documentation channels
Google Drive MCP Google Drive Read SME notes, design docs, and product requirement documents

Agents — Two Autonomous Documentation Workflows

Agents orchestrate Skills and MCP connections to run complete workflows autonomously. I define the goal and review the output. The agent handles everything in between.

Agent Trigger What It Does Autonomously My Job
Release Notes Pipeline GUS story closed with doc-impact tag Reads story → classifies → runs Skill → checks consistency → commits to GitHub → notifies Slack Review PR in 10 minutes
Weekly Feedback Triage Every Monday 8 AM Reads Drive feedback → categorizes → deduplicates → prioritizes top 5 → creates GitHub stories → fixes top issue → posts Slack summary Review summary in 20 minutes

The Complete Pipeline

MONDAY MORNING — WEEKLY FEEDBACK TRIAGE Google Drive (MCP) → Reads last week's feedback doc → doc-consistency-auditor SKILL runs → Deduplicates + prioritizes top 5 issues → GitHub MCP: creates 5 sprint stories → Filesystem + GitHub MCP: fixes #1 issue → Slack MCP: posts health summary → Writer reviews in 20 minutes ✓ ──────────────────────────────────── CONTINUOUS — RELEASE NOTES PIPELINE GitHub (MCP) → Detects closed GUS story (doc-impact tag) → release-note-generator SKILL runs → Consistency check via Filesystem MCP → GitHub MCP: commits to release notes file → Slack MCP: posts review notification → Writer approves PR in 10 minutes ✓

What I Learned Building This

01

Skills are the highest-leverage investment

Writing a good Skill takes 2–3 hours. But it pays back immediately — every task that uses it is faster, more consistent, and requires less review.

02

MCP removes the biggest friction

Before MCP, significant time was spent copying content into Claude and out again. MCP eliminated that entirely — Claude reads and writes the real files.

03

Quality gates are more important than capabilities

The most important design decision in each agent was what it could not do without human approval. Every agent ends with a human review step.

04

This is systems architecture, not writing

Building this shifted my thinking. I am not a writer who uses AI tools. I am a documentation systems architect who designs pipelines that produce documentation.
